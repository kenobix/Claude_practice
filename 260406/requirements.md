# 個人開発管理ダッシュボード 要件定義書

作成日: 2026-04-06  
作成方法: Claude Code サブエージェント（技術調査・機能要件・非機能要件を並列実施）

---

## 1. システムコンセプト

WSL上で動作し、スマートフォンからもアクセスできる個人開発者向けの生産性管理ツール。
タスク管理・時間記録・マークダウンノートを一元化し、どこからでも作業状況を把握・更新できる。

---

## 2. 採用技術スタック

| 技術 | 採否 | 用途・理由 |
|------|------|-----------|
| **SQLite** | 採用 | ローカルDB。個人用途に十分、バックアップが `cp` 一発 |
| **Cloudflare Tunnel** | 採用 | スマホアクセスの核心。無料枠で利用可、ポート開放不要 |
| **Firebase Auth** | 採用（限定） | Google認証によるスマホからの本人確認（Spark無料プラン） |
| **Docker** | 採用 | `docker compose up -d` 1コマンド起動のため |
| PostgreSQL | スキップ | 個人用途に過剰。WSL再起動のたびにサービス管理が煩雑 |
| Cloudflare Workers/D1 | スキップ | ローカル完結構成には不要 |
| Firebase Firestore/Storage | スキップ | SQLiteと役割が重複、コスト増のみ |

### 追加採用技術

| 技術 | 用途 |
|------|------|
| Hono (Node.js v20 LTS) | バックエンドAPI |
| React + Vite | フロントエンド |
| Drizzle ORM | DBスキーマ管理・マイグレーション |
| Tailwind CSS | UIスタイリング |

## 3. システム構成図

```
スマートフォン / ブラウザ
    │ HTTPS (dashboard.yourdomain.com)
    ▼
Cloudflare Edge（TLS終端・DDoS保護）
    │ 暗号化トンネル
    ▼
WSL2 (Ubuntu)
    ├── cloudflared（トンネルデーモン）Docker コンテナ
    ├── Hono / Node.js :3000（REST API + 静的ファイル配信）
    └── SQLite（~/dashboard/data/app.db）

認証フロー:
スマホ → Firebase Auth（Google認証）→ JWT取得
→ バックエンドへリクエスト時に Authorization ヘッダーで送信
→ firebase-admin SDK でトークン検証
```

## 4. 機能要件

### 4.1 ユーザーストーリー

| # | ストーリー |
|---|-----------|
| US-01 | タスクを素早く追加したい。作業中に思いついたことをすぐに記録するため。 |
| US-02 | タスクをプロジェクト単位でグループ化したい。何に取り組んでいるかを俯瞰するため。 |
| US-03 | タスクに優先度とステータスを設定したい。今日やるべきことを明確にするため。 |
| US-04 | スマートフォンからタスクを確認・更新したい。外出中でも作業状況を把握するため。 |
| US-05 | タスクに対して計測を開始・停止したい。何時間費やしているかを知るため。 |
| US-06 | 日次・週次の作業時間サマリーを見たい。生産性パターンを把握するため。 |
| US-07 | マークダウンでノートを書きたい。コードスニペットや手順を整形して保存するため。 |
| US-08 | ノートをタスクやプロジェクトに紐付けたい。文脈のある知識として蓄積するため。 |
| US-09 | ノートをキーワード検索したい。過去に調べたことを素早く再参照するため。 |

### 4.3 画面構成

```
/                    ダッシュボード（今日のタスク・稼働タイマー・直近ノート）
├── /tasks           タスク一覧（フィルタ・ソート・ステータス変更）
│   └── /tasks/:id   タスク詳細・編集・タイマー操作
├── /projects        プロジェクト一覧
│   └── /projects/:id プロジェクト詳細（タスク・時間・ノート集約）
├── /notes           ノート一覧（検索・タグ絞り込み）
│   ├── /notes/new   ノート新規作成
│   └── /notes/:id   ノート詳細・編集（Markdownエディタ＋プレビュー）
├── /timer           アクティブタイマー確認・停止
└── /reports         時間集計・週次グラフ
```

```
Project 1 ── N Task 1 ── N TimeLog
    │              │
    └── N Note ────┘
          │
          N Tag（多対多）
```

| エンティティ | 主要フィールド |
|------------|--------------|
| Project | id, name, description, color, status, created_at |

## 5. 非機能要件

### 5.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| API応答（読み取り系） | 200ms以内 |
| API応答（書き込み系） | 500ms以内 |
| ページ初期ロード（ローカル） | 1秒以内 |
| ページ初期ロード（スマホ経由） | 2秒以内 |

### 5.2 セキュリティ

- Firebase Auth による JWT 認証必須（インターネット公開のため）
- シークレットは `.env` 管理、**Git コミット禁止**
- DB は Docker ネットワーク外に公開しない
- Cloudflare Tunnel により TLS 終端（証明書管理不要）

### 5.3 運用・起動

- **全サービスの起動は `docker compose up -d` 1コマンド**
- WSL 自動起動: `/etc/wsl.conf` の `[boot] command` で設定
- バックアップ: 1日1回・7世代保持（cronで自動化）
- `GET /health` ヘルスチェックエンドポイントを実装

### 5.4 移植性

- ホストに必要なツールは `docker`・`docker compose`・`git` のみ
- Linux 環境であれば WSL 以外でも同一手順で起動できること
- ファイル I/O は Linux ファイルシステム（`/home/` 配下）に配置

| サービス | プラン | 個人利用コスト |

| SQLite | オープンソース | 無料 |
| Docker | Personal | 無料 |

| Firebase Auth | Spark（無料）プラン | 無料（月1万認証まで） |
| Hono / React / Vite / Drizzle | オープンソース | 無料 |
| **合計** | | **ほぼ無料（ドメイン代のみ）** |

> **注記**: スマホアクセスに独自ドメインが必要（Cloudflare で年 $10 程度）。ローカルネットワーク内のみのアクセスで良ければドメインも不要で完全無料。
