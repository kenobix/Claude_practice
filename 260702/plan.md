# 生成AI連携ブロックチェーン実装計画

作成日: 2026-07-02
使用API: Gemini API（無料枠）

## ゴール
1. ブロックチェーンの基本的な仕組みを自分の手で理解する
2. 最終的に生成AI（Gemini API）と連携するブロックチェーンプロダクトを作る

---

## フェーズ1: 自作ブロックチェーンで仕組みを理解する（目安1〜2週間）

スクラッチ実装を通じて「ブロックチェーンが何を保証し、何を保証しないか」を体感する。
生成AIはペアプロ相手（設計相談・実装補助・テストコード生成）として活用する。

### 実装項目
- ブロック構造の定義（hash, prevHash, timestamp, data, nonce）
- ハッシュチェーンによる改ざん検知の仕組み
- 簡易PoW（マイニング）の実装
- トランザクションプールの実装
- 簡易署名検証（ECDSA）
- 簡易P2P同期（複数ノード間でのチェーン共有、最長チェーン優先ルール）

### 技術スタック案
- 言語: Python または TypeScript
- 外部ライブラリは最小限（ハッシュ計算・署名検証程度）

---

## フェーズ2: 既存チェーンへの橋渡し（スマートコントラクト入門）

フェーズ1の理解を土台に、実用性のある既存チェーン上での開発に移行する。

### 実装項目
- ローカルテストネット環境の構築（Hardhat / Anvil など）
- ERC-20トークンの実装・デプロイ
- ERC-721（NFT）の実装・デプロイ
- フェーズ1で学んだ概念（ハッシュチェーン・署名・合意）と実チェーンの対応関係を整理

### 技術スタック案
- Solidity + Hardhat（Ethereum系）
- ローカルテストネットでの動作確認

---

## フェーズ3: 生成AI API連携プロダクト（AIオラクル型）

### 採用パターン: AIオラクル型
オフチェーンでGemini APIがデータを解析・判定し、その結果をハッシュ化してオンチェーンに記録する構成。
生成AIの推論結果に「改ざん耐性」と「証跡（プロブナンス）」を与えることが目的。

```
[入力データ] → [Gemini APIで解析・判定] → [結果をハッシュ化]
                                              ↓
                                    [スマートコントラクトに記録]
                                              ↓
                              [誰でも後から検証可能な証明として利用]
```

### 実装項目
- Gemini API（無料枠）を用いた解析・判定ロジックの実装
  - 例: 文書の真贋判定、画像やテキストの分類、査定コメント生成 など
- 解析結果のハッシュ化（SHA-256等）
- ハッシュとメタデータ（タイムスタンプ、入力の要約等）をオンチェーンに記録するコントラクト
- 検証用フロント（記録されたハッシュと再計算したハッシュを比較し、改ざんの有無を確認）

### プロダクト案（例）
- AI生成物の来歴・権利証明サービス（生成コンテンツのハッシュをオンチェーンに記録）
- AI審査付き証明書発行（Geminiが書類やデータを審査し、結果をオンチェーン証明化）

### 技術スタック案
- Gemini API（無料枠、レート制限に注意）
- フェーズ2で構築したスマートコントラクト基盤を再利用
- バックエンド: Python または Node.js（Gemini API呼び出し・ハッシュ計算）

---

## 補足: Gemini API無料枠についての留意点
- レート制限（リクエスト数/分、トークン数等）があるため、デモ規模の設計にとどめる
- 無料枠の仕様は変更されることがあるため、実装直前に最新のクォータを確認する
